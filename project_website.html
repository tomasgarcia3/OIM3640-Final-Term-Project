The Crypto Trading Bot

Big Idea:

The idea behind this project was to create a crypto trading bot that would collect data on a crypto coin, then analyze its metrics and finally create a buy or sell order based on the trading strategy.

Goal:

My main goal with this project was not only to create a profitable trading bot, but also being able to learn more about python and improve my skills as well as understanding of the features and frameworks.

Why did I do this:

The main reason why I chose to build a crypto trading bot as my final project was because I like to keep up and invest in financial markets and always though how insteresting it would be to explore the possibilities of a trading bot. Before this experience I had never interacted with a trading bot so it made me realize many things about using trading bots to make money. On one end, I now understand the complexity that comes with profitable trading bots and the enormous amount of time that has to be put into it. I spent so much time in this project and it does not even come close to being completely profitable. However although I found this complex an challenging, I can see why many banks and financial entities are investing so much money into trading algorithms and bots because of how efficient they are to collect and process data.

User Instructions

This crypto trading bot uses 100% python. It uses the CCXT library to get information about ticker, used TA-LIB library for the analysis matrics, and Gemini Sandbox exchange to place orders.

Steps to follow:

1. Get Gemini Sandbox API

Follow the link below to create an account and get the API with Gemini Sandbox.

  • https://exchange.sandbox.gemini.com/

2. Install all dependencies

TA-Lib Library:

  • https://pypi.org/project/TA-Lib/

Pandas:

  • https://pypi.org/project/pandas/

Time:

3. Download bot.py

Once you have installed all dependencies you can download the bot.

4. Update API with your own

Then you would need to update the api with your own public and private key.

5. Run bot with desired ticker

You can run the bot to make money/or lose. Good luck!

Implementation Information

Bot.py is composed of 4 functions:

1. Collects data

The first function uses CCXT library to collect information from ticker.

2. Analyzes the data

Afterwards, this function take sin the data and obtains RSI which is used to determine if the coin should be bought based on whether it was oversold or overbought.

3. Executes trade

The order function would then get the recommendation and either buy, sell or wait based on what is recommended in the analysis.

4. Putting everything together

Finally this function puts everything together and in the desired order. It will also communicate each time the type of recommendation given.Keep in mind that if the previous recommendation is the same it will not be executed again. Meaning that if the exchange bought the ticker it will not buy it again if the recommendation to buy is given a second time in a row.

Results

Waiting to place order

The software is running and will not place a buy or sell order unless is recommended by the RSI trading strategy.

Placing a BUY order

Once the RSI strategy recommends to buy the ticker then the order will be executed and recorded.

Keeping order

As you can see the program recognizes a BUY order has already been executed and thus it will not buy more unless a SELL order is places.

Placing a SELL order

The program waits until the RSI trading strategy recommends to sell the order.

Project Evolution

The creation was not easy. It involved a lot of learning about using the different libraries and existing programs for this bot to work. I initially thought that it was not going to be too hard, but I was mistaken.

The process started by looking at different existing bots. Most were written in python but even others in languages like JavaScript. In this step, I saw many approaches that I considered taking, however, in the first attempts I ended up finding barriers that were very hard to pass. Some involved deep learning of Pandas and NumPy which I did not have. Others involved deep technical skills with respect to analysis and algorithms which I couldn't even understand. With this said, I had to start over many times. This is when I decided to take a more simple approach and use libraries to facilitate the development of this program.

Once I entered the mentality of simplicity and learned about different libraries I was able to find the way to build this program. First of all, I found inspiration in the structure of this program from one of the resources listed below. This allowed me to figure out that the program could just be an integration of 4 functions in which one collects the data, the second analyzes the data, the third executed the order, and the fourth just brings the program together. Once I figured this out I began by collecting the data from the CCXT library. I knew the data I needed depended on the purpose of analysis so I also dug deeper into TA-Lib and determined I would use RSI as my indicator. I chose RSI because it was a term I knew from before and seemed relatively simple to determine when the crypto was overbought or oversold (The links to finding more information about these two libraries and the ways they can be used are below). Then I proceeded to incorporate the Gemini Sandbox API into the program to execute the order based on the buy or sell signal. For this, I also used the resource on their website to find more information about how to create this order and get information from the ticker using the API. I finally put everything together and got the desired result.

I want to say that this process was hard and I failed many times in the process. I took different approaches along the way to find barriers that would not let me continue. It was frustrating at times to have to start over again but I am glad I did not quit because I feel like I learned a lot along the way. The result was possible through a lot of investigation and learning found in the resources listed below. With respect to the profitability of my bot, I want to say that it still relies on a lot of luck. Even though the example above shows a profit made from buying and selling in some cases before and after the trade came out to a loss. So while I was unable to make a profitable bot, I still got all the learning experiences I wanted out of this assignment.
